# cc-deck Style Guide

## Voice
- Professional, thorough, no contractions
- Terminal-native analogies
- Context before commands, reasoning-first explanations
- Use cc-deck voice profile (`.style/voice.yaml`)

## Terminology
- Use "podman" exclusively (never Docker)
- Capitalize Kubernetes resources: Pod, Service, Deployment
- Use monospace for field names: `spec.replicas`
- "environment" not "env" in prose (abbreviation OK in CLI context)

## AsciiDoc
- One sentence per line (semantic line breaks)

## Formatting
- Never use em-dash or en-dash characters
- Use commas, parentheses, or separate sentences instead
