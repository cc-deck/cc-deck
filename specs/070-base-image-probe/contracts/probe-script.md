# Contract: Probe Shell Script

The probe script is a self-contained POSIX shell script generated by Go code and executed inside the target base image via `podman run --rm --entrypoint /bin/sh <image> -c "<script>"`.

## Input

The script takes no arguments. The tool list is baked into the generated script.

## Output

One JSON object per line on stdout. Three categories of output lines:

### OS Info Line
```json
{"type":"os","id":"fedora","id_like":"","name":"Fedora Linux 41","version":"41"}
```

### Package Manager Line
```json
{"type":"pkgmgr","name":"dnf","path":"/usr/bin/dnf"}
```

### Tool Line (one per probed tool)
```json
{"type":"tool","name":"git","path":"/usr/bin/git","version":"2.43.0","present":true}
{"type":"tool","name":"go","path":"","version":"","present":false}
```

### User Info Line
```json
{"type":"user","name":"root","uid":0,"home":"/root","shell":"/bin/bash"}
```

### Shell Availability Line
```json
{"type":"shells","available":["bash","sh"]}
```

## Error Handling

- Lines that are not valid JSON are silently ignored (stderr noise from the container)
- If `which` is not available, the script falls back to checking `/usr/bin/<tool>` and `/usr/local/bin/<tool>` directly
- Individual tool detection failures do not abort the script; the tool is reported as `present: false`

## Timeout

The probe script must complete within the configured timeout (default 30s). The Go caller enforces this via `context.WithTimeout`. If the timeout is exceeded, the entire probe is considered failed and the build falls back to defaults.

## Version Extraction

For each tool, the script tries these patterns in order:
1. `<tool> --version 2>&1 | head -1`
2. `<tool> version 2>&1 | head -1`
3. `<tool> -v 2>&1 | head -1`

The Go parser extracts the first semver-like pattern (`\d+\.\d+(\.\d+)?`) from the output line.
