# Implementation Plan: Plugin Lifecycle Management

**Branch**: `009-plugin-lifecycle` | **Date**: 2026-03-04 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/009-plugin-lifecycle/spec.md`

## Summary

Add `plugin install`, `plugin status`, and `plugin remove` subcommands to the cc-deck Go CLI. The compiled Zellij plugin WASM binary is embedded in the CLI at build time via `//go:embed`. Installation writes the WASM to `~/.config/zellij/plugins/`, creates a layout file, and optionally injects the plugin pane into the user's default Zellij layout. Status reports installation state with Zellij version compatibility. Remove undoes all changes cleanly.

## Technical Context

**Language/Version**: Go 1.22+ (existing project uses Go 1.25 in go.mod)
**Primary Dependencies**: cobra (CLI), go:embed (binary embedding), os/exec (Zellij detection)
**Storage**: Filesystem only (WASM binary, KDL layout files)
**Testing**: `go test` with filesystem-based tests using `t.TempDir()`
**Target Platform**: macOS, Linux (local CLI)
**Project Type**: CLI (extension to existing cc-deck)
**Performance Goals**: N/A (all operations are single-shot file I/O)
**Constraints**: Offline-capable, single binary distribution
**Scale/Scope**: 3 new subcommands, 1 new package, approximately 600-800 lines of new Go code
