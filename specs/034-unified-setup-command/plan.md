# Implementation Plan: Unified Setup Command

**Branch**: `034-unified-setup-command` | **Date**: 2026-04-08 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/034-unified-setup-command/spec.md`

## Summary

Unify the existing `cc-deck image` command (container image building) and SSH remote provisioning into a single `cc-deck setup` command with a shared manifest (`cc-deck-setup.yaml`), two Claude Code slash commands (`/cc-deck.capture` and `/cc-deck.build`), and Ansible playbooks as the SSH provisioning backend. The existing `internal/build` package is renamed to `internal/setup` and extended with a `Targets` config supporting both container and SSH targets. The SSH pre-flight bootstrap (`internal/ssh/bootstrap.go`) is replaced by a lightweight probe in `env create --type ssh`.

## Technical Context

**Language/Version**: Go 1.25 (from go.mod)
**Primary Dependencies**: cobra v1.10.2 (CLI), gopkg.in/yaml.v3 (YAML), adrg/xdg (XDG paths via internal wrapper), go:embed (command embedding), os/exec (ansible-playbook execution)
**Storage**: YAML files in `.cc-deck/setup/` (project-local), generated Ansible artifacts alongside manifest
**Testing**: `make test` (Go tests), manual testing against SSH target (Hetzner VM)
**Target Platform**: macOS (development), Linux (provisioning targets)
**Project Type**: CLI tool
**Performance Goals**: N/A (interactive CLI workflow)
**Constraints**: Ansible must be installed locally for SSH targets. Initial SSH target support limited to Fedora/RHEL (dnf).
**Scale/Scope**: Single developer, single target host per SSH section
