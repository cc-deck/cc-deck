# Manifest Schema Contract: cc-deck-setup.yaml

## Schema Version

Version 2. Version 1 manifests (`cc-deck-image.yaml` with `image:` field) are not auto-migrated. Users create new manifests via `cc-deck setup init`.

## Structure

The manifest has two conceptual halves:

1. **Shared sections** (target-agnostic): `tools`, `sources`, `plugins`, `mcp`, `github_tools`, `settings`, `network`
2. **Target sections**: `targets.container`, `targets.ssh`

### Invariants

- The `/cc-deck.capture` command MUST only modify shared sections. It MUST NOT read or write target sections.
- The `/cc-deck.build` command reads shared sections and the relevant target section. It MUST NOT modify the manifest.
- Both target sections can coexist. Presence of one does not affect the other.
- A manifest with no target sections is valid (capture can populate shared sections before targets are added).

### Field Semantics

#### tools ([]string)

Free-form human-readable tool descriptions. Examples: `"go 1.25"`, `"ripgrep"`, `"Python 3.12"`.

The build command maps these to concrete install actions (dnf packages, downloads, etc.) using AI interpretation. The manifest does not enforce package names or versions.

#### github_tools ([]GithubTool)

Tools downloaded from GitHub Releases. The `asset_pattern` field supports `{arch}` placeholder for architecture substitution (maps to target architecture: `aarch64`, `x86_64` for Linux).

#### settings.shell_rc (string)

Path to the curated shell config file. Relative paths are resolved from the setup directory. The file is generated by `/cc-deck.capture` and stored in the setup directory (typically `./build-context/zshrc`).

#### targets.container.registry (string)

Optional. If set, enables `--push` on `/cc-deck.build --target container`. Format: `quay.io/org` or `ghcr.io/org`. The full push reference is `<registry>/<name>:<tag>`.

#### targets.ssh.create_user (bool)

If true, the Ansible `base` role creates the user specified in `targets.ssh.user` with sudo access and SSH authorized keys from the current user's `~/.ssh/authorized_keys` or the key specified in `targets.ssh.identity_file`.

#### targets.ssh.workspace (string)

Remote directory for workspace creation. Default: `~/workspace`. Tilde is expanded on the remote, not locally (resolves F-002).

### Compatibility

- Version field must be >= 1
- Unknown fields are ignored (forward compatibility)
- Missing optional fields use documented defaults
- The manifest filename is `cc-deck-setup.yaml` (replaces `cc-deck-image.yaml`)
