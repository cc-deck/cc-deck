## Implementation Phases

### Phase 1: Core CLI Restructure

Rename command files, update command registration, create config parent. This is the foundational change that everything else depends on.

**Files**:
- Rename `env.go` → `ws.go`: Change `NewEnvCmd` → `NewWsCmd`, set `Use: "ws"`, `Aliases: []string{"workspace"}`, rename subcommand groups
- Rename subcommands: `create` → `new` (Use field), `delete` → `kill` (Use field)
- Rename `env_promote.go` → `ws_promote.go`: Keep attach/list, add exec (extract `newExecCmdCore`), remove status/start/stop/logs exports
- Rename `setup.go` → `build.go`: Change `NewSetupCmd` → `NewBuildCmd`, set `Use: "build"`
- Create `config.go`: New `NewConfigCmd` grouping plugin, profile, domains, completion
- Move `newCompletionCmd` from main.go to `internal/cmd/` as `NewCompletionCmd`
- Update `main.go`: New group definitions, updated command registration

**Key technical details**:
- `newExecCmdCore` extraction: Current `newEnvExecCmd` (env.go:1479) needs splitting into core + env wrapper, following the attach/list pattern
- `addToGroup` helper: Keep duplicated (main.go and ws.go), both are package-private
- Promoted top-level: `ls` is the primary Use with `list` as alias (swap from current); `attach` and `exec` keep their names

### Phase 2: Internal Package Rename

Rename `internal/setup/` → `internal/build/` and update manifest filename.

**Files**:
- Rename directory `internal/setup/` → `internal/build/`
- Update `cc-deck-setup.yaml` → `cc-deck-build.yaml` in manifest constants
- Update all import paths referencing `internal/setup`
- Update help text and error messages referencing "setup"
