# Implementation Plan: Controller Leader Election

**Branch**: `055-controller-leader-election` | **Date**: 2026-05-14 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/055-controller-leader-election/spec.md`

## Summary

Fix navigation mode chaos and voice indicator flicker caused by duplicate WASM controller instances (Zellij bug). Add a ping-pong leader election protocol so only one controller is active, then restore the `broadcast_render_all` safety net for reliable sidebar rendering.

## Technical Context

**Language/Version**: Rust stable (edition 2021), compiled to wasm32-wasip1
**Primary Dependencies**: zellij-tile 0.43.1, serde/serde_json 1.x
**Storage**: WASI `/cache/` directory (sessions.json, debug.log)
**Testing**: `cargo test` (unit tests, non-WASM), `make test`, `make lint`
**Target Platform**: WASM (wasm32-wasip1) running inside Zellij 0.44.x
**Project Type**: Zellij plugin (background controller + sidebar renderer)
**Performance Goals**: No perceptible latency added by election protocol
**Constraints**: Election must complete within 2 seconds; steady-state overhead <= 2 pipe messages per minute
**Scale/Scope**: 10-15 tabs, 2 controller instances (Zellij bug), 10-15 sidebar instances

## Design Decisions

### 1. Dormant guard placement

The dormant guard is a simple early return at the top of `pipe()` and `update()` methods. This is the least invasive approach: all existing logic remains unchanged, it just doesn't execute when `is_leader == false`.

```
pipe():
  if !is_leader && name != "cc-deck:controller-ping" && name != "cc-deck:controller-pong":
    return false

update():
  if !is_leader && !matches!(event, PermissionRequestResult | Timer):
    return false
  // Timer is needed for election ticks and re-activation checks
```

The Timer event is still processed when dormant, but only for election-related logic (counting ticks, checking re-activation timeout). All other timer work (flush render, poll git, etc.) is skipped.

### 2. Ping payload format

Use the existing `cc-deck:controller-ping` pipe name with the sender's `plugin_id` as the string payload. The receiver parses it as `u32`. This is the simplest format that carries the only information needed for election (who sent the ping).

### 3. Keybinding registration timing

Currently, keybindings are registered on the first `TabUpdate` event. With the election protocol, this is deferred: the leader registers keybindings after the election completes (2-second probe window). This means keybindings are unavailable for ~2 seconds on fresh Zellij start, which is acceptable because Zellij's own UI is still initializing during this period.

### 4. broadcast_render_all restoration

Restore the function as it existed before commit 8ff10c0, called at the end of `broadcast_render()` after the targeted sends. With only one active controller, the untargeted broadcast is safe and provides a fallback for sidebars not yet in the registry.
