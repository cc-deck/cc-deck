# Implementation Plan: Centralize Workspace Definitions

**Branch**: `041-centralize-workspace-definitions` | **Date**: 2026-04-22 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/041-centralize-workspace-definitions/spec.md`

## Summary

Centralize all workspace definitions into a single file (`~/.config/cc-deck/workspaces.yaml`), replace project-local `.cc-deck/workspace.yaml` files with git-committable templates (`.cc-deck/workspace-template.yaml`), and simplify default workspace resolution to a two-phase lookup (project-dir ancestor match, then global recency). This eliminates the dual-source definition model, the project registry in state.yaml, and the per-project status files.

## Implementation Phases

### Phase 1: Foundation (P1 - Central Store + Templates + Resolution)

This phase delivers all three P1 user stories in a bottom-up order: data layer first, then command layer.

#### Task 1.1: Template Types and Parser

**New file**: `internal/ws/template.go` + `internal/ws/template_test.go`

- Define `WorkspaceTemplate` struct (Name string, Variants map[string]TemplateVariant)
- Define `TemplateVariant` as a wrapper around `WorkspaceDefinition` fields (using YAML unmarshaling into a map, then mapping to WorkspaceDefinition)
- `LoadTemplate(projectRoot string) (*WorkspaceTemplate, error)`: reads `.cc-deck/workspace-template.yaml`
- `ExtractPlaceholders(yamlData []byte) []Placeholder`: regex scan for `{{name}}` and `{{name:default}}`
- `ResolvePlaceholders(yamlData []byte, answers map[string]string) []byte`: string substitution
- `PromptForPlaceholders(placeholders []Placeholder) (map[string]string, error)`: interactive prompting with defaults
- `ValidateTemplate(tmpl *WorkspaceTemplate) error`: checks name present, variants non-empty, variant keys valid

**Tests**: Load valid template, missing name error, unknown variant type error, placeholder extraction, default value handling, resolution substitution.

**Acceptance**: FR-002, FR-003, FR-004, FR-005

#### Task 1.2: Extend DefinitionStore

**Modify**: `internal/ws/definition.go` + `internal/ws/definition_test.go`

- Add `FindByProjectDir(path string) ([]*WorkspaceDefinition, error)`: loads all definitions, returns those where `path` is at or below `def.ProjectDir` (ancestor match using `filepath.Rel` or `strings.HasPrefix` on canonical paths).
- Add `AddWithCollisionHandling(def *WorkspaceDefinition) (string, error)`: checks for name conflict; if same name + same type, returns error; if same name + different type, appends `-{type}` suffix to name, then adds. Returns the final name used.

**Tests**: FindByProjectDir with exact match, subdirectory match, no match; AddWithCollisionHandling with no collision, same-type collision (error), different-type collision (auto-suffix).

**Acceptance**: FR-001, FR-008, FR-009 (collision handling)

#### Task 1.3: Rewrite Default Resolution

**Modify**: `internal/cmd/ws.go` - `resolveWorkspaceName` function

Replace the current filesystem-walk implementation (lines 1897-1932) with:

1. If positional arg provided, return it (unchanged)
2. Create `DefinitionStore` and `FileStateStore`
3. Get cwd, call `defs.FindByProjectDir(cwd)` for ancestor-matching definitions
4. If matches found:
   - One match: return its name
   - Multiple: look up `WorkspaceInstance.LastAttached` for each, select most recent
5. If no matches: fall back to `defs.List(nil)` (all definitions)
   - One: return its name
   - Multiple: select by most recent `LastAttached`
   - Multiple with no attachment: error with guidance
   - None: error with guidance
6. Print `Using workspace "X"` to stderr (FR-007)

Remove: project auto-registration (`store.RegisterProject`), gitignore self-healing call.

**Tests**: Resolution with single workspace, multiple with recency, project-scoped selection, no workspaces error, multiple with no history error.

**Acceptance**: FR-006, FR-007

#### Task 1.4: Rewrite `ws new` Command

**Modify**: `internal/cmd/ws.go` - `runWsNew` function

Major changes to the RunE handler:

1. **Template loading**: After determining project root, try `LoadTemplate(projectRoot)`. If found:
   - Select variant by `--type` or auto-select if single
   - Extract and prompt for placeholders
   - Resolve placeholders into variant fields
   - Build `WorkspaceDefinition` from variant
   - Apply CLI flag overrides (changed flags take precedence)
2. **Name resolution**: Use template name, explicit arg (overrides template), or directory basename
3. **Central storage**: Call `defs.AddWithCollisionHandling(def)` instead of `SaveProjectDefinition`
4. **Set project-dir**: `def.ProjectDir = project.CanonicalPath(cwd)` for all types
5. **Remove**: scaffolding, `--global`/`--local` flags, `ProjectStatusStore` writes, `store.RegisterProject`, cross-project collision check via `AllProjectWorkspaceNames`
6. **Remove `newFlags` fields**: `global`, `local`
7. **Remove** `MarkFlagsMutuallyExclusive("global", "local")`
8. **Preserve**: all type-specific option setting, repo handling, image/storage resolution, the Create() call

**Tests**: Create from template with placeholders, create without template (pure flags), collision handling, explicit name overriding template name, single-variant auto-selection, multi-variant without --type error.

**Acceptance**: FR-001, FR-002, FR-003, FR-004, FR-005, FR-008, FR-009, FR-013

#### Task 1.5: Update `ws list`

**Modify**: `internal/cmd/ws.go` - `runWsList`, `writeWsTableWithProjects`, `buildSourceMap`, structured output types

**Tests**: List with project associations, list with "-" for standalone workspaces.
