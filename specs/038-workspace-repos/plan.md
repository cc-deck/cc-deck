# Implementation Plan: Remote Workspace Repository Provisioning

**Branch**: `038-workspace-repos` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/038-workspace-repos/spec.md`

## Summary

Add automatic git repository cloning to remote workspace environments during `env create`. Developers declare repos in their environment definition or pass them via `--repo` flags. The system clones repos in parallel (max 4), supports auto-detection of the current git repo, and handles credential transport via SSH agent forwarding (SSH envs) or token injection with post-clone cleanup (all env types). Works across SSH, container, compose, and k8s-deploy environment types using a common `CommandRunner` abstraction over each type's exec mechanism.

## Implementation Phases

### Phase 1: Core Data Model and Repo Logic

**Goal**: RepoEntry type, URL normalization, clone command generation, and unit tests.

**Files**:
- `cc-deck/internal/env/repos.go` (NEW): RepoEntry struct, `NormalizeURL()`, `RepoNameFromURL()`, `buildCloneCommand()`, `cloneRepos()` function with CommandRunner
- `cc-deck/internal/env/repos_test.go` (NEW): Tests for URL normalization, repo name extraction, clone command generation, deduplication
- `cc-deck/internal/env/definition.go` (MODIFY): Add `Repos []RepoEntry` field

**Key design**:
```go
type CommandRunner func(ctx context.Context, cmd string) (string, error)

func resolveGitCredentials(credType GitCredentialType, credSecret string) (*GitCredentials, error)

func cloneRepos(ctx context.Context, runner CommandRunner, repos []RepoEntry, workspace string, creds *GitCredentials, extraRemotes map[string]string) []RepoCloneResult
```

The `resolveGitCredentials` function resolves the token value from an env var or K8s Secret reference. Returns nil for SSH type (handled by agent forwarding at the SSH client level) or when unconfigured.

The `cloneRepos` function:
1. Deduplicates repos by normalized URL
2. For each repo (parallel, max 4):
   a. Check if `<workspace>/<target>/` exists via runner
   b. If exists, skip
   c. Build clone command (with token injection if needed)
   d. Execute via runner
   e. If branch specified, checkout
   f. If token was used, rewrite origin URL
   g. If extraRemotes provided and this is the auto-detected repo, add each remote via `git remote add`
3. Return results (success/warning per repo)

The `extraRemotes` parameter is a separate `map[string]string` (name -> URL), not part of RepoEntry. This keeps the data model simple and avoids overloading RepoEntry with auto-detection-specific concerns.

### Phase 3: Environment Type Integration

**Goal**: Wire cloneRepos into each environment's Create() method.

**Files**:
- `cc-deck/internal/env/ssh.go` (MODIFY): After provisioning, call `cloneRepos` with SSH runner. Set `AgentForwarding=true` on SSH client when Profile has SSH credentials.
- `cc-deck/internal/env/container.go` (MODIFY): After container start, call `cloneRepos` with podman exec runner.
- `cc-deck/internal/env/compose.go` (MODIFY): After compose up, call `cloneRepos` with compose exec runner.
- `cc-deck/internal/env/k8s_deploy.go` (MODIFY): After pod ready, call `cloneRepos` with kubectl exec runner.
- `cc-deck/internal/podman/exec.go` (MODIFY): Add `ExecOutput()` variant that returns stdout as string.

Each environment's Create() passes:
- Its own CommandRunner wrapping the internal exec mechanism
- The resolved workspace path (from definition or type default)
- Git credentials resolved from the active Profile

### Phase 4: CLI Flags and Auto-Detection

**Goal**: Add `--repo`/`--branch` flags and auto-detect current git repo.

**Files**:
- `cc-deck/internal/cmd/env.go` (MODIFY):
  - Add `repos []string` and `branches []string` fields to `createFlags`
  - Register `--repo` (repeatable StringArrayVar) and `--branch` (repeatable StringArrayVar) flags
  - Parse into `[]RepoEntry` with branch-to-repo association
  - Validate `--branch` requires preceding `--repo`
  - Auto-detect: if in git repo, read remotes via `git remote -v`, build RepoEntry for origin
  - Merge definition repos + CLI repos + auto-detected repo
  - Pass merged list to environment Create via the definition
  - Set Zellij working directory to auto-detected repo's workspace path

### Phase 5: Local Environment Warning

**Files**:
