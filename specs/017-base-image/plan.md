# Implementation Plan: cc-deck Base Container Image

**Branch**: `017-base-image` | **Date**: 2026-03-12 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/017-base-image/spec.md`

## Summary

Create a Fedora-based container image that serves as the foundation for Claude Code
development environments. The image provides a comprehensive developer toolbox (30+
CLI tools, Node.js, Python/uv, zsh with starship) but deliberately excludes Zellij,
Claude Code, and cc-deck (added during user image build for version consistency).
Published as multi-arch (amd64 + arm64) to `ghcr.io/rhuss/cc-deck-base`.

## Technical Context

**Language/Version**: Containerfile (OCI image build), shell scripts (bash)
**Primary Dependencies**: Fedora 41 base image, dnf packages, starship (GitHub release)
**Storage**: N/A (stateless image artifact)
**Testing**: Shell-based verification script (tool availability checks)
**Target Platform**: Linux containers (amd64 + arm64)
**Project Type**: Container image (Containerfile + build scripts)
**Performance Goals**: Image size < 1.5 GB compressed, all tools available within 5s of shell start
**Constraints**: Multi-arch support required, no cc-deck/Zellij/CC in base image
**Scale/Scope**: Single image, ~30 tools, 2 architectures
