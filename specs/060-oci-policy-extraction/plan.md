# Implementation Plan: OCI Policy Extraction

**Branch**: `060-oci-policy-extraction` | **Date**: 2026-05-23 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/060-oci-policy-extraction/spec.md`

## Summary

Extract policy files directly from OCI images at runtime for OpenShell sandbox creation, replacing the current host-path dependency. Uses a two-phase approach: build-time label stamping records which layer contains the policy file, and runtime extraction fetches that layer (or falls back to a full scan) to obtain the policy without requiring the build directory on the host.

## Design Decisions

### D1: Package Boundary

The `internal/oci/` package provides three public functions:
- `ExtractFileFromImage(imageRef, filePath string) ([]byte, error)` - Main entry point for runtime extraction
- `FindLayerContaining(img v1.Image, filePath string) (v1.Hash, error)` - Layer scanning utility
- `AddLabel(imageRef, key, value string) error` - Label injection for build-time

This keeps OCI concerns isolated from workspace and build logic.

### D2: Label Key Convention

Label key: `dev.cc-deck.policy-layer`
Label value: Layer diff ID as `sha256:<hex>`

The `dev.cc-deck` prefix follows OCI reverse-DNS convention. The diff ID format matches what `go-containerregistry` returns from `layer.DiffID()`.

### D3: Extraction Strategy

1. Parse image reference
2. Try local daemon first (`daemon.Image`), fall back to remote (`remote.Image`)
3. Read config labels for `dev.cc-deck.policy-layer`
4. If label found: fetch that specific layer, extract file from tar
5. If label missing or file not in labeled layer: reverse-scan all layers
6. Return file bytes or error

### D4: Build-Time Integration Point

After `runOpenShellBuild()` completes the podman build (line ~356) but before push (line ~373):
1. Load the built image from local daemon
2. Call `FindLayerContaining` to locate the policy layer
3. Call `AddLabel` to stamp the digest
4. The labeled image is then available for push

### D5: Runtime Integration Point

In `resolveSandboxConfig()` (openshell.go:104-127):
- If `def.Policy` is empty (no explicit policy path set by user)
- And `def.SandboxImage` is set
- Call `ExtractFileFromImage(def.SandboxImage, "/etc/openshell/policy.yaml")`
- Write bytes to temp file, set `cfg.Policy` to temp path
- Defer cleanup of temp file in the calling `Create()` method

### D6: Error Handling

- Registry unreachable: Return error with message suggesting `--policy` manual override
- No policy file found in any layer: Return error with same suggestion
- Label points to nonexistent layer: Log warning, fall back to full scan
- Auth failure: Return error from credential chain (go-containerregistry provides descriptive errors)
