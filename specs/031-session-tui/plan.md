# Implementation Plan: Session TUI (Control Plane Dashboard)

**Branch**: `031-session-tui` | **Date**: 2026-03-30 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/031-session-tui/spec.md`

## Summary

Build a full-screen terminal UI (`cc-deck tui`) that serves as a control plane for all cc-deck environments. The TUI provides a live, auto-refreshing environment list with status polling, attach via suspend/resume, a create wizard for local and container environments, lifecycle management (start/stop/delete), and a help overlay. P1 uses direct polling (no daemon). Built with bubbletea v0.25+ and the Charm stack, integrated as a subcommand of the existing `cc-deck` binary.

## Technical Context

**Language/Version**: Go 1.25 (from go.mod)
**Primary Dependencies**: cobra v1.10.2 (CLI), bubbletea v0.25+ (TUI), lipgloss v1.x (styling), bubbles (components), internal/env (environment management)
**Storage**: `$XDG_STATE_HOME/cc-deck/state.yaml` (existing), `~/.config/zellij/plugins/cc_deck.wasm/cache/sessions.json` (read-only)
**Testing**: `go test` via `make test`
**Target Platform**: macOS, Linux (terminal emulators with 256-color support)
**Project Type**: CLI tool (new subcommand in existing binary)
**Performance Goals**: TUI launches and displays environments within 2 seconds (SC-001); polling does not block UI rendering
**Constraints**: No daemon in P1; bubbletea v0.25+ required for suspend/resume; build via Makefile only
**Scale/Scope**: Supports 10+ environments (SC-005); P1 covers local + container types only

**Structure Decision**: Single new package `internal/tui` with all TUI code. The TUI imports `internal/env` and `internal/cmd` but nothing imports TUI. This keeps the dependency graph clean and ensures the TUI is an optional feature of the binary.
