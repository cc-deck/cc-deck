# Implementation Plan: Network Security and Domain Filtering

**Branch**: `022-network-filtering` | **Date**: 2026-03-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/022-network-filtering/spec.md`

## Summary

Add domain-based network filtering to cc-deck containerized sessions. For Podman deployments, generate a tinyproxy sidecar on an internal network that restricts egress to allowed domains. For K8s/OpenShift, refactor existing NetworkPolicy and EgressFirewall generation to use a new domain group expansion system. Domain groups are composable (built-in groups embedded in the binary, user-defined groups in `~/.config/cc-deck/domains.yaml`), referenced by name in manifests, and expandable at deploy time.

## Implementation Phases

### Phase 1: Domain Group System (P1 foundation)

Core domain expansion engine. No deployment integration yet.

1. **`internal/network/builtin.go`**: Define built-in domain groups as Go map literals. Groups: `python`, `nodejs`, `rust`, `golang`, `github`, `gitlab`, `docker`, `quay`. Backend groups: `anthropic`, `vertexai`.

2. **`internal/network/config.go`**: Load `~/.config/cc-deck/domains.yaml` using adrg/xdg + gopkg.in/yaml.v3. Non-fatal if missing. Parse `extends`, `includes`, `domains` fields per group.

3. **`internal/network/domains.go`**: Core expansion logic:
   - `ExpandGroup(name string) ([]string, error)`: Resolve a group name to domain list
   - `ExpandAll(names []string) ([]string, error)`: Expand multiple names, merge, dedup
   - `WildcardDedup(domains []string) []string`: Remove entries covered by wildcards
   - Recursive includes with cycle detection (visited set)
   - Group vs domain disambiguation (no dot = group, has dot = domain)
   - Merge built-in + user config (user overrides or extends built-in)

4. **`internal/network/domains_test.go`**: Unit tests covering:
   - Built-in group expansion
   - User group override (no extends)
   - User group extension (extends: builtin)
   - Recursive includes
   - Cycle detection
   - Wildcard dedup
   - Unknown group name error
   - Mixed group names and literal domains

### Phase 2: CLI Commands (P2 discoverability)

5. **`internal/cmd/domains.go`**: New `cc-deck domains` command with subcommands:
   - `init`: Seed domains.yaml with commented built-in definitions
   - `list`: Show all groups with source annotation
   - `show <group>`: Display expanded domains with source per entry

### Phase 3: Manifest Integration (P1 deploy)

6. **`internal/build/manifest.go`**: Add `NetworkConfig` struct to Manifest:
   ```go
   type NetworkConfig struct {
       AllowedDomains []string `yaml:"allowed_domains,omitempty"`
   }
   ```
   Add `Network *NetworkConfig` field to Manifest struct.

7. **`internal/cmd/deploy.go`**: Add `--allowed-domains` flag with `+`/`-`/bare syntax parsing. Merge with manifest's `network.allowed_domains`.

### Phase 4: Compose Generation (P1 Podman deploy)

8. **`internal/compose/proxy.go`**: Generate tinyproxy config from expanded domain list:
   - `tinyproxy.conf` with `FilterDefaultDeny Yes`, `FilterType fnmatch`
   - `whitelist` file with one domain pattern per line
   - Wildcard conversion: `.example.com` becomes `*.example.com` for fnmatch

9. **`internal/compose/generate.go`**: Generate `compose.yaml` from manifest:
   - Session container on internal network only
   - Proxy container on internal + default networks
   - `HTTP_PROXY`/`HTTPS_PROXY` env vars pointing to proxy
   - Volume mount for proxy config
   - Volume mount for proxy access log (for `domains blocked`)
   - `.env.example` with credential placeholders

### Phase 5: K8s/OpenShift Refactor (P3)

10. **`internal/k8s/network.go`**: Refactor `backendDNSNames()` and `backendEgressRules()` to use domain group expansion instead of hardcoded switch statements. Feed expanded domain lists through existing `AllowedEgress` parameter. Preserve existing behavior (no functional change for current users).

### Phase 6: Audit and Runtime (P3)

11. **`internal/cmd/domains.go`**: Add `blocked` subcommand. Parse tinyproxy access log from proxy container via `podman logs` or mounted log volume.

12. **`internal/cmd/domains.go`**: Add `add` and `remove` subcommands. Regenerate proxy whitelist and restart proxy container (not session container) via compose project name convention.

### Phase 7: Extract Integration (P2)

13. **`internal/build/commands/cc-deck.extract.md`**: Update the AI command template to detect project ecosystem files and add domain groups to manifest:
   - `go.mod` -> `golang`
   - `pyproject.toml`, `requirements.txt`, `.python-version` -> `python`
   - `package.json`, `.nvmrc` -> `nodejs`
   - `Cargo.toml` -> `rust`

| Risk | Mitigation |
|------|-----------|

| Breaking existing --allow-egress users | Keep --allow-egress as alias for --allowed-domains during transition |
| Compose generation is entirely new code path | Comprehensive unit tests for generated YAML structure |
