# Implementation Plan: Environment Interface and CLI

**Branch**: `023-env-interface` | **Date**: 2026-03-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/023-env-interface/spec.md`

## Summary

Define a unified Environment interface for cc-deck that abstracts lifecycle management (create, start, stop, delete, attach, status) across local, Podman, and K8s backends. Implement the `cc-deck env` CLI command group with 12 subcommands, a versioned `state.yaml` file at `$XDG_STATE_HOME`, the Local environment implementation, backward-compatible aliases for existing commands, and automatic migration of legacy config.yaml sessions.

## Technical Context

**Language/Version**: Go 1.25 (from go.mod)
**Primary Dependencies**: cobra v1.10.2 (CLI), adrg/xdg v0.5.3 (XDG paths), gopkg.in/yaml.v3 (YAML), client-go v0.35.2 (K8s, existing)
**Storage**: YAML file at `$XDG_STATE_HOME/cc-deck/state.yaml` (new), `$XDG_CONFIG_HOME/cc-deck/config.yaml` (existing, migration source)
**Testing**: `go test` via `make test`, table-driven tests
**Target Platform**: macOS (darwin), Linux
**Project Type**: CLI tool (Go)
**Performance Goals**: List < 1s, create+attach < 3s, detailed status < 5s
**Constraints**: No external dependencies beyond what exists in go.mod. Zellij >= 0.40 required for local environments.
**Scale/Scope**: Dozens of environments per user (not hundreds)
