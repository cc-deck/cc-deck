# Implementation Plan: cc-deck Sidebar Plugin

**Branch**: `012-sidebar-plugin` | **Date**: 2026-03-07 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/012-sidebar-plugin/spec.md`

## Summary

cc-deck v2 replaces the failed dual-mode plugin approach with a focused sidebar plugin for Claude Code session management in Zellij. The sidebar appears on every tab (via layout tab_template), shows real-time Claude activity indicators, and supports click-to-switch, attend (jump to waiting), rename, and session creation. A Go CLI handles installation (with safe settings.json backup), hook forwarding, and plugin lifecycle.

## Technical Context

**Language/Version**: Rust stable (edition 2021, wasm32-wasip1 target) for plugin; Go 1.22+ for CLI
**Primary Dependencies**: zellij-tile 0.43.1 (plugin SDK), serde/serde_json 1.x; cobra (CLI), encoding/json (Go stdlib)
**Storage**: WASI `/cache/` directory for plugin state; filesystem for installation artifacts
**Testing**: `cargo test` (native, not WASM) for Rust; `go test` for Go
**Target Platform**: WASM (wasm32-wasip1) for plugin; Darwin/Linux for CLI
**Project Type**: Terminal plugin + CLI tool (monorepo)
**Performance Goals**: Hook command <100ms exit time; sidebar render <16ms; state sync <1s across instances
**Constraints**: WASM sandbox (no direct process execution, no shared memory); Zellij plugin API surface
**Scale/Scope**: 2-10 concurrent Claude sessions per Zellij session; 1-5 Zellij sessions

## Implementation Phases

### Phase 1: Foundation (P1 - Core Sidebar + Hooks)

Build the minimum viable plugin: sidebar renders sessions, hook events flow in, install works.

**Milestone**: User can install cc-deck, start Claude sessions, and see activity in the sidebar.

1. **Rust Plugin Core** (state.rs, session.rs, config.rs, main.rs)
   - PluginState with session tracking (BTreeMap<u32, Session>)
   - Activity enum with state transitions
   - Plugin load/update/pipe/render lifecycle
   - Configuration parsing from KDL (sidebar width, mode)
   - Permission request handling

2. **Pipe Handler** (pipe_handler.rs)
   - Parse cc-deck:hook messages (JSON payload)
   - Map hook events to Activity state transitions
   - SessionEnd handling (remove session)

3. **Sidebar Rendering** (sidebar.rs)
   - Vertical session list with activity indicators
   - Active tab highlighting
   - Elapsed time display
   - Name truncation with ellipsis
   - Overflow indicators (scroll support)
   - Empty state display
   - Mouse click handling (click-to-switch)

4. **Multi-Instance Sync** (sync.rs)
   - Broadcast state on changes (cc-deck:sync)
   - Request state on load (cc-deck:request)
   - Merge incoming state (newest timestamp wins)
   - Pane-to-tab mapping refresh from local events

5. **Git Detection** (git.rs)
   - Async `run_command()` for git repo and branch detection
   - Handle RunCommandResult events
   - Auto-naming from repo name or directory basename

6. **Go CLI: Hook Command** (hook.go)
   - Read stdin JSON, parse hook payload
   - Read ZELLIJ_PANE_ID from environment
   - Forward via `zellij pipe`
   - Silent failure on all error conditions

7. **Go CLI: Install/Uninstall** (install.go, remove.go, hooks.go, backup.go)
   - WASM copy to plugins directory
   - Layout file generation and installation
   - settings.json backup (timestamped)
   - Hook registration/removal in settings.json
   - Idempotent operation
   - --skip-backup flag

8. **Layout File** (embedded in Go CLI)
   - KDL layout with tab_template including sidebar
   - Compact-bar at bottom for tab navigation

### Phase 2: Navigation + Management (P2 - Attend, Rename, Create)

Add keyboard-driven workflow features.

**Milestone**: User can jump to waiting sessions, rename sessions, and create new ones from the sidebar.

1. **Attend Action** (attend.rs)
   - Scan sessions for Waiting state
   - Find oldest waiting session
   - switch_tab_to() for focus
   - Inline notification when no sessions waiting

2. **Notification System** (notification.rs)
   - Brief flash message in sidebar
   - Auto-dismiss after timeout
   - Timer-based rendering

3. **Session Rename** (rename.rs)
   - Inline text input in sidebar
   - Key event handling (typing, Enter, Escape)
   - Update display_name + rename_tab()
   - Duplicate name resolution (numeric suffix)

4. **Session Creation**
   - open_command_pane() with claude command
   - Auto-name from git detection
   - Register in session state on CommandPaneOpened event

5. **Keybinding Registration** (config.rs additions)
   - reconfigure() for attend, rename, new-session keybindings
   - Configurable key assignments

## Risk Mitigation

| Risk | Mitigation |
|------|-----------|
| Permission dialog invisible (v1 failure) | Sidebar pane has enough vertical space for permission UI |
| State sync loops (rename_tab triggers TabUpdate) | Guard flag to prevent re-entrancy (zellaude pattern) |
| settings.json corruption (zellaude failure) | Go's encoding/json + mandatory backup before modify |
| WASM sandbox limitations | All I/O through run_command() and pipe APIs |
| Too many sidebar instances (10+ tabs) | Pipe broadcast is O(n) but n is small; state is compact JSON |
