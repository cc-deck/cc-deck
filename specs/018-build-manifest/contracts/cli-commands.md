# CLI Contract: cc-deck build commands

## `cc-deck build init <dir>`

**Purpose**: Scaffold a new build directory.

| Aspect | Detail |
|--------|--------|
| Args | `<dir>` (required): target directory path |
| Flags | `--force`: overwrite existing build directory |
| Exit 0 | Directory created with all scaffolded files |
| Exit 1 | Directory already contains `cc-deck-build.yaml` (without --force) |
| Output | Lists created files |

## `cc-deck build <dir>`

**Purpose**: Build the container image from the manifest and Containerfile.

| Aspect | Detail |
|--------|--------|
| Args | `<dir>` (required): build directory path |
| Flags | `--tag <tag>`: override manifest tag |
| Exit 0 | Image built successfully |
| Exit 1 | Missing Containerfile, invalid manifest, or build failure |
| Output | Image name:tag, size, build duration |

**Pre-conditions**:
1. `cc-deck-build.yaml` exists and is valid
2. `Containerfile` exists (generated by `/cc-deck.containerfile`)
3. Container runtime (podman or docker) is available

**Self-embedding**: Copies its own binary to `.build-context/cc-deck` before build.

## `cc-deck push <dir>`

**Purpose**: Push the built image to a registry.

| Aspect | Detail |
|--------|--------|
| Args | `<dir>` (required): build directory path |
| Flags | `--registry <url>`: override registry from manifest |
| Exit 0 | Image pushed successfully |
| Exit 1 | Image not built, push failure, auth failure |
| Output | Registry URL, pushed tags |

## `cc-deck build verify <dir>`

**Purpose**: Smoke-test a built image.

| Aspect | Detail |
|--------|--------|
| Args | `<dir>` (required): build directory path |
| Flags | none |
| Exit 0 | All checks pass |
| Exit 1 | One or more checks fail |
| Output | Pass/fail for: tool availability, Claude Code startup, cc-deck version |

## `cc-deck build diff <dir>`

**Purpose**: Show changes since last Containerfile generation.

| Aspect | Detail |
|--------|--------|
| Args | `<dir>` (required): build directory path |
| Flags | none |
| Exit 0 | Diff displayed (or no changes) |
| Exit 1 | Missing manifest or Containerfile |
| Output | Added/removed/changed tools, plugins, MCP entries |

## Manifest Schema (`cc-deck-build.yaml`)

See `data-model.md` for the full schema definition.

Required fields: `version`, `image.name`
All other sections are optional.
