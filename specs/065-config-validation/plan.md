# Implementation Plan: Config Validation

**Branch**: `065-config-validation` | **Date**: 2026-06-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/065-config-validation/spec.md`

## Summary

Add a `cc-deck config check` command and load-time validation for `config.yaml`. Validation lives in the `internal/config/` package as a `Config.Validate()` method that returns a slice of `Finding` structs, following the existing `Profile.Validate()` pattern. The CLI command prints findings with severity, category, message, and fix suggestion. Load-time integration prints a one-line summary to stderr when issues are found.

## Technical Context

**Language/Version**: Go 1.25 (from go.mod)
**Primary Dependencies**: cobra v1.10.2 (CLI), gopkg.in/yaml.v3 (config parsing), unicode (stdlib, for East Asian Width property)
**Storage**: YAML config file at `~/.config/cc-deck/config.yaml` (read-only, via `internal/xdg`)
**Testing**: `go test` via `make test`
**Target Platform**: macOS, Linux (cross-platform CLI)
**Project Type**: CLI tool
**Performance Goals**: Validation completes in <100ms, load-time overhead <5ms
**Constraints**: No external dependencies for Unicode width analysis (use Go stdlib `unicode` package)
**Scale/Scope**: Single config file, ~50 badge rules max, ~10 profiles max
