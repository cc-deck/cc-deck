# Implementation Plan: Plugin Bugfixes

**Branch**: `010-plugin-bugfixes` | **Date**: 2026-03-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/010-plugin-bugfixes/spec.md`

## Summary

Fix five issues in the cc-deck Zellij plugin: (1) dynamically update tab titles with status icons and project names via `rename_tab()`, (2) auto-detect manually started Claude sessions by scanning `PaneInfo.title` on `PaneUpdate` events, (3) auto-start Claude in new sessions using `open_command_pane_in_place`, (4) render the session picker as a floating overlay using `open_command_pane_floating`, and (5) create an automated integration test script.

## Technical Context

**Language/Version**: Rust (stable, wasm32-wasip1 target) with zellij-tile 0.43.1
**Primary Dependencies**: zellij-tile 0.43 (plugin SDK), serde/serde_json (serialization)
**Storage**: WASI `/cache/` directory for persistent state (recent.json)
**Testing**: `cargo test` for unit tests, shell script for integration tests
**Target Platform**: WASM (wasm32-wasip1) running inside Zellij 0.40+
**Project Type**: Zellij plugin (WASM binary)
**Performance Goals**: Tab title updates within 1 second, session detection within 5 seconds
**Constraints**: WASI sandbox (no direct filesystem access), single-row status bar pane
**Scale/Scope**: 5 fixes to existing plugin, approximately 200-400 lines of Rust changes
