# Quickstart: cc-deck setup run

## Prerequisites

- Existing setup directory with artifacts generated by `/cc-deck.build`
- For container targets: podman or docker installed
- For SSH targets: ansible-playbook installed (`pip install ansible`)

## Basic Usage

### Rebuild a Container Image

```bash
# From the project directory (auto-detects .cc-deck/setup/)
cc-deck setup run

# Or specify the setup directory explicitly
cc-deck setup run .cc-deck/setup/
```

### Re-provision an SSH Host

```bash
# Auto-detects SSH target from site.yml + inventory.ini
cc-deck setup run
```

### Build and Push a Container Image

```bash
# Build, then push to the registry configured in the manifest
cc-deck setup run --push
```

### Explicit Target Selection

When both container and SSH artifacts exist in the same setup directory:

```bash
cc-deck setup run --target container
cc-deck setup run --target ssh
```

## Workflow Context

```
cc-deck setup init          # Scaffold setup directory
/cc-deck.capture            # Discover tools and config (Claude Code)
/cc-deck.build              # Generate artifacts (Claude Code)
cc-deck setup run           # Execute the build (direct CLI) ← this command
cc-deck setup verify        # Smoke-test the result
cc-deck setup diff          # Check for manifest drift
```
