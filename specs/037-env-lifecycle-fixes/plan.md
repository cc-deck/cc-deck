# Implementation Plan: Environment Lifecycle Fixes

**Branch**: `037-env-lifecycle-fixes` | **Date**: 2026-04-14 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/037-env-lifecycle-fixes/spec.md`

## Summary

Fix three environment lifecycle issues: (1) type resolution when an explicit name differs from the project-local definition, (2) SSH delete not cleaning up definitions, (3) missing source indicator in environment list. Additionally, add `--global`/`--local` flags as an escape hatch for definition resolution, remove the PATH column from list output, and show project path in status output.

## Implementation Phases

### Phase 1: SSH Delete Cleanup (FR-005, FR-006)

**Rationale**: Simplest change, self-contained, provides immediate user value.

**Changes**:
- `cc-deck/internal/env/ssh.go`: Add `defs.Remove()` call in `Delete()` method, matching the pattern from `ContainerEnvironment.Delete()` (container.go:362-366)
- `cc-deck/internal/env/ssh_test.go`: Add test verifying definition removal after delete

**Pattern to follow** (from container.go:362-366):
```go
if e.defs != nil {
    if err := e.defs.Remove(e.name); err != nil {
        log.Printf("WARNING: removing definition: %v", err)
    }
}
```

### Phase 2: Type Resolution Fix (FR-001, FR-002, FR-002a, FR-003, FR-004)

**Rationale**: Core bug fix. Depends on understanding the existing resolution chain.

**Changes to `cc-deck/internal/cmd/env.go` in `runEnvCreate()`**:

1. After name resolution (line 216) and before type resolution (line 222), add a new decision branch:
   - If explicit name was provided AND `projDef != nil` AND `name != projDef.Name`:
     - Set `projDef = nil` (ignore project-local definition for this create)
     - Look up `defs.FindByName(name)` for a global definition
     - If found: use global definition's type and settings (populate a "resolved definition" variable)
     - If not found: fall back to `local` type (FR-003)
   - Set a flag `usedGlobalDef = true` to suppress scaffolding at line 241

2. Guard scaffolding (line 241): skip when `usedGlobalDef` is true

3. Settings application (lines 259-301): apply from the resolved definition (global or project-local) rather than always from `projDef`

**Tests** (`cc-deck/internal/cmd/env_create_test.go`):
- Test: explicit name matching global def uses global type
- Test: explicit name not matching any def falls back to local
- Test: explicit name matching project-local def uses project-local (regression)
- Test: `--type` overrides global definition type

### Phase 3: `--global` and `--local` Flags (FR-012, FR-013, FR-014, FR-015)

**Rationale**: Builds on Phase 2's resolution refactoring. Adds escape hatch.

**Changes to `cc-deck/internal/cmd/env.go`**:

1. Add `global` and `local` fields to `createFlags` struct
2. Register flags in `newEnvCreateCmd()` with `cmd.MarkFlagsMutuallyExclusive("global", "local")`
3. In `runEnvCreate()`, before the resolution chain:
   - If `--global`: require `defs.FindByName(name)` succeeds, set `projDef = nil`, use global definition
   - If `--local`: require `projDef != nil`, ignore global definitions
   - Both flags error if the required definition does not exist

**Tests**:
- Test: `--global` selects global definition over project-local
- Test: `--local` selects project-local over global
- Test: `--global` with missing global def returns error
- Test: `--local` with missing project-local def returns error
- Test: `--global --local` returns mutual exclusion error
- Test: `--global --type` uses global settings with overridden type

### Phase 4: List SOURCE Column and PATH Removal (FR-007, FR-008, FR-010)

**Rationale**: UI changes that are independent of the create/delete fixes.

**Changes to `cc-deck/internal/cmd/env.go`**:

1. Add `Source string` field to `envListEntry` struct with `json:"source" yaml:"source"` tags
2. In `writeEnvStructured()`: set `Source` to `"global"` for definition-sourced entries, `"project"` for project entries, empty for orphan instances
3. For instances: determine source by checking if name exists in global definitions or in a project-local definition
4. In `writeEnvTableWithProjects()`: replace PATH column with SOURCE column
5. Remove PATH from the table header and row formatting

**Tests**:
- Test: table output contains SOURCE column, no PATH column
- Test: JSON output includes `"source"` field with correct values

### Phase 5: Status Project Path (FR-009)

**Rationale**: Complements Phase 4 by moving path information to status.

**Changes to `cc-deck/internal/cmd/env.go`**:

1. Add `ProjectPath string` field to `envStatusOutput` struct with `json:"project_path,omitempty" yaml:"project_path,omitempty"` tags
2. In `runEnvStatus()`: look up the environment name in the project registry (iterate `store.ListProjects()`, load each project's definition, match by name)
3. In `writeEnvStatusText()`: print `Project:` line when project path is non-empty

**Tests**:
- Test: status output includes project path for project-local environment
- Test: status output omits project path for global environment

### Phase 6: Documentation and Cleanup

2. Update CLI reference (`docs/modules/reference/pages/cli.adoc`) with `--global`/`--local` flags
