# Implementation Plan: Single Binary Merge

**Branch**: `031-single-binary-merge` | **Date**: 2026-04-01 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/031-single-binary-merge/spec.md`

## Summary

Merge the two WASM binaries (`cc_deck_controller.wasm` + `cc_deck_sidebar.wasm`) into a single `cc_deck.wasm` that determines its role from KDL configuration at runtime. The primary goal is solving the background plugin permission dialog problem in Zellij 0.44, with secondary benefits of simplified build pipeline and install flow. The approach uses a `UnifiedPlugin` enum wrapper that delegates to the existing `ControllerPlugin` or `SidebarRendererPlugin` based on `configuration.get("mode")` at load time.

## Technical Context

**Language/Version**: Rust stable (edition 2021, wasm32-wasip1 target) for plugin; Go 1.25 for CLI
**Primary Dependencies**: zellij-tile 0.44 (plugin SDK), serde/serde_json 1.x; cobra (CLI), encoding/json (Go stdlib)
**Storage**: WASI `/cache/` directory for persistent state (sessions.json)
**Testing**: `cargo test` (native target with WASM stubs), `go test` (Go CLI)
**Target Platform**: WASM (wasm32-wasip1) running inside Zellij terminal multiplexer
**Project Type**: Terminal plugin (Rust WASM) + CLI tool (Go)
**Performance Goals**: Sidebar instances must not receive heavy events (PaneUpdate, TabUpdate, Timer) to preserve N-instance scaling
**Constraints**: Single WASM binary must serve both headless controller and UI sidebar roles
**Scale/Scope**: One plugin binary, ~10 Go files to update, ~1 new Rust file (UnifiedPlugin)

## Implementation Phases

### Phase 1: Rust Plugin (Single Binary)

**Goal**: Produce a single `cc_deck.wasm` that dispatches to controller or sidebar at runtime.

**Changes**:

1. **Cargo.toml**: Remove `[features]` section and both `[[bin]]` targets. Add single `[[bin]]` target:
   ```toml
   [[bin]]
   name = "cc_deck"
   path = "src/main.rs"
   ```

2. **src/main.rs**: 
   - Remove `#[cfg(feature = "controller")]` and `#[cfg(feature = "sidebar")]` gates on module declarations
   - Always declare both `mod controller;` and `mod sidebar_plugin;`
   - Add `UnifiedPlugin` enum:
     ```rust
     enum UnifiedPlugin {
         Controller(controller::ControllerPlugin),
         Sidebar(sidebar_plugin::SidebarRendererPlugin),
         Uninitialized,
     }
     ```
   - Implement `ZellijPlugin` for `UnifiedPlugin`, delegating all methods to the active variant
   - In `load()`: read `configuration.get("mode")`, initialize the appropriate variant, call its `load()`
   - Default to `Sidebar` when mode is absent
   - `register_plugin!(UnifiedPlugin)`
   - Remove the legacy no-feature-flag registration path (lines 357-456)

3. **Event subscriptions**: Each variant's `load()` already calls `subscribe()` with the correct event set. No changes needed within controller or sidebar modules.

**Validation**: `cargo test` passes, `cargo build --target wasm32-wasip1 --release` produces single `cc_deck.wasm`.

### Phase 2: Build System (Makefile)

**Goal**: Single build target, cleanup old binaries.

**Changes**:

1. **Simplify variables**:
   - Remove `WASM_CTRL_SRC`, `WASM_CTRL_DST`, `WASM_SIDE_SRC`, `WASM_SIDE_DST`
   - Keep `WASM_SRC` and `WASM_DST` pointing to single binary

2. **Replace targets**:
   - Remove `build-wasm-controller` and `build-wasm-sidebar`
   - Add single `build-wasm` target: `cargo build --target wasm32-wasip1 --release`
   - Update `copy-wasm` to copy single binary
   - Add cleanup step in `install` target to remove old `cc_deck_controller.wasm` and `cc_deck_sidebar.wasm` from plugins directory

3. **Update `build-wasm-debug`** to match (no feature flags)

**Validation**: `make build-wasm` produces `cc_deck.wasm`, `make install` succeeds.

### Phase 3: Go CLI (Embed + Install)

**Goal**: Simplify to single embedded binary, remove permission workaround.

**Changes**:

1. **embed.go**:
   - Remove `cc_deck_controller.wasm` and `cc_deck_sidebar.wasm` embeds
   - Keep single `//go:embed cc_deck.wasm` embed
   - Simplify `PluginInfo`: single `Binary []byte` and `BinarySize int64`
   - Remove `ControllerBinary`, `SidebarBinary`, `ControllerSize`, `SidebarSize`

2. **install.go**:
   - Replace two-binary atomic write with single `atomicWrite()` call
   - Remove rollback logic (only one file to write)
   - Remove `EnsureControllerPermissions()` call
   - Remove legacy `cc_deck.wasm` cleanup (it IS the binary now)
   - Remove old `cc_deck_controller.wasm` and `cc_deck_sidebar.wasm` if present

3. **zellij.go**:
   - Remove `EnsureControllerPermissions()` function entirely

4. **layout.go**:
   - Update `sidebarPluginBlock()`: reference `cc_deck.wasm` instead of `cc_deck_sidebar.wasm`
   - Update `controllerConfigBlock()`: reference `cc_deck.wasm` instead of `cc_deck_controller.wasm`

5. **remove.go**:
   - Simplify to remove single `cc_deck.wasm`
   - Keep controller config removal from config.kdl (still needed)

6. **state.go**:
   - Update `InstallState` to check for `cc_deck.wasm` instead of `cc_deck_controller.wasm`

7. **layout_test.go**:
   - Update test assertions for new binary name references

**Validation**: `go test ./internal/plugin/...` passes.

### Phase 4: Integration Testing

**Goal**: Verify end-to-end functionality.

1. `make install` succeeds with single binary
2. Single `cc_deck.wasm` in `~/.config/zellij/plugins/`
3. No `cc_deck_controller.wasm` or `cc_deck_sidebar.wasm` present
4. Layout references `cc_deck.wasm` with `mode "sidebar"`
5. config.kdl references `cc_deck.wasm` with `mode "controller"`
6. Fresh Zellij session: permission dialog appears on sidebar, granting covers controller
7. Session tracking, hook processing, navigation, rename, delete all functional

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Zellij deduplicates same-URL instances | Low | High | Maintainer suggested this pattern; pending explicit confirmation on zellij-org/zellij#4982 |
| Permission cache not shared across configs | Low | High | Maintainer confirmed cache is keyed by URL |
| Binary size regression | Low | Low | Both modules share most code; combined size should be ~900KB vs current ~830KB |
| Legacy code (sync.rs, old PluginState) increases binary size | Low | Low | Out of scope for this feature; cleanup is separate |
| `register_plugin!` macro incompatible with enum | Low | Medium | Verify macro expansion; enum implements `Default` + `ZellijPlugin` |
