# Implementation Plan: Single-Instance Architecture

**Branch**: `030-single-instance-arch` | **Date**: 2026-03-31 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/030-single-instance-arch/spec.md`

## Summary

Split the current single WASM plugin (one instance per tab) into two binaries: a background controller (single instance, owns all state and event processing) and a thin sidebar renderer (one per tab, displays cached payloads, handles local interaction). This eliminates O(N) event processing overhead, removes the entire synchronization subsystem (~527 LOC), and makes sidebar responsiveness independent of tab count.

## Technical Context

**Language/Version**: Rust stable (edition 2021, wasm32-wasip1 target) for plugin; Go 1.25 for CLI
**Primary Dependencies**: zellij-tile 0.43.1 (plugin SDK), serde/serde_json 1.x; cobra (CLI), encoding/json (Go stdlib)
**Storage**: WASI `/cache/` directory for persistent state (sessions.json)
**Testing**: `cargo test` (native target with WASM stubs), `go test`
**Target Platform**: WASM (wasm32-wasip1) for plugins, native for CLI
**Project Type**: Terminal multiplexer plugin + CLI tool
**Performance Goals**: Constant sidebar response time regardless of tab count (2 tabs = 20 tabs)
**Constraints**: Plugin-to-plugin pipe targeting via plugin_url is broken in Zellij (config matching issue). Must use broadcast+filter pattern.
**Scale/Scope**: 10-30 concurrent tabs with active Claude Code sessions

## Implementation Phases

### Phase 1: Shared Types and Build Infrastructure

Extract shared types to `lib.rs`, set up two-binary build with feature flags. No behavioral changes yet.

**Key changes:**
- Cargo.toml: Add `controller` and `sidebar` features, two `[[bin]]` targets
- lib.rs: Move Session, Activity, HookPayload, config types. Add RenderPayload, ActionMessage, SidebarHello, SidebarInit types
- main.rs: cfg-gated ZellijPlugin registration
- Makefile: Two WASM build targets (`build-wasm-controller`, `build-wasm-sidebar`)

**Verification:** Both binaries compile, `cargo test` passes, existing functionality unchanged.

### Phase 2: Controller Plugin

Implement the background controller that owns state and processes events.

**Key changes:**
- controller/mod.rs: ZellijPlugin impl (load subscribes to heavyweight events, pipe handles hooks+actions)
- controller/state.rs: ControllerState with sessions BTreeMap, sidebar_registry
- controller/events.rs: Handlers for PaneUpdate, TabUpdate, Timer, RunCommandResult, PaneClosed
- controller/hooks.rs: Hook event processing (extracted from current pipe handler)
- controller/actions.rs: Process ActionMessage from sidebars (switch, rename, delete, pause, attend)
- controller/render_broadcast.rs: Build RenderPayload from state, broadcast with coalescing
- controller/sidebar_registry.rs: Hello/init handshake, tab reindexing on TabUpdate

**Verification:** Controller processes events, builds render payloads. Unit tests for state transitions.

### Phase 3: Sidebar Plugin

Implement the thin renderer that displays cached payloads and handles local interaction.

**Key changes:**
- sidebar/mod.rs: ZellijPlugin impl (load subscribes to Mouse+Key only, pipe handles render+init)
- sidebar/state.rs: SidebarState with cached_payload, mode, click_regions
- sidebar/render.rs: ANSI rendering from RenderPayload (adapted from current sidebar.rs)
- sidebar/input.rs: Mouse click handling (session selection, rename trigger) and key handling (navigate, filter)
- sidebar/modes.rs: SidebarMode state machine (extracted from current state.rs)
- sidebar/rename.rs: Inline rename editing (adapted from current rename.rs)

**Verification:** Sidebar renders from payload, handles input, sends actions. Mode transition tests.

### Phase 4: Go CLI Updates

Update the CLI to embed both binaries, generate correct layouts, and route hooks to controller.

**Key changes:**
- embed.go: Two `go:embed` directives, updated PluginInfo with ControllerBinary/SidebarBinary
- install.go: Write both WASM files, handle migration from single binary
- remove.go: Remove both WASM files
- layout.go: Add `load_plugins` block for controller in generated KDL
- hook.go: Add `--plugin` flag targeting controller binary in `zellij pipe` invocation
- status.go: Report both binaries in status output

**Verification:** `cc-deck plugin install` writes both binaries, layouts include controller loading, hooks target controller.

### Phase 5: Sync Elimination and Cleanup

Remove the entire synchronization subsystem. Clean up dead code paths.

**Key changes:**
- Delete sync.rs entirely
- Remove sync-related fields from state (sync_dirty, pending_overrides, last_meta_content_hash)
- Remove sync integration points from event handlers (~20 callsites)
- Remove session-meta.json read/write
- Remove shared last_click file (double-click detection moves to sidebar-local)
- Remove broadcast_and_save, merge_sessions, flush_if_dirty
- Simplify controller persistence to single-writer pattern

**Verification:** All sync code removed, `cargo test` passes, no sync-related references remain.

### Phase 6: Integration Testing and Documentation

End-to-end verification and documentation updates.

**Key changes:**
- Integration tests: Controller+sidebar interaction via simulated pipe messages
- Protocol roundtrip tests: Serialize/deserialize all message types
- README.md: Update architecture description, add spec 030 to feature table
- Documentation: Update any references to single-plugin architecture
- Makefile: Verify `make install`, `make test`, `make lint` all work

**Verification:** Full test suite passes, `make install` produces working two-binary setup, documentation current.
