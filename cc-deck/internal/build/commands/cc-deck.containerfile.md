description: Generate a Containerfile from the cc-deck build manifest

## User Input

$ARGUMENTS

## Outline

Generate a complete, valid Containerfile from `cc-deck-build.yaml`. The Containerfile is always regenerated from scratch (never patched).

### Step 1: Read the manifest

Read `cc-deck-build.yaml` and validate it has at least `version` and `image.name`.

### Step 2: Resolve tools to install commands

For each entry in the `tools` section, determine the concrete install command:

- Map to `dnf install` packages where possible (the base image is Fedora)
- For tools not in Fedora repos, use GitHub release downloads or language-specific installers
- For version constraints (e.g., ">= 1.23"), choose the appropriate installation method
- Handle multi-arch support (use `${TARGETARCH}` for GitHub release downloads)

### Step 3: Generate the Containerfile

Write a Containerfile with this structure:

```dockerfile
# GENERATED BY cc-deck.containerfile - DO NOT EDIT
# Regenerate with: claude /cc-deck.containerfile

FROM <base-image>

# Layer: System packages (changes rarely)
USER root
RUN dnf install -y <packages> && dnf clean all

# Layer: Language-specific tools (changes occasionally)
RUN <language tool installs>

# Layer: GitHub tools (cc-setup, cc-session, etc.)
RUN <curl downloads for each github_tool>

# Layer: cc-deck self-install (Zellij + plugin + layouts)
COPY cc-deck /usr/local/bin/cc-deck
RUN cc-deck plugin install --install-zellij --force

# Layer: Claude Code
RUN npm install -g @anthropic-ai/claude-code

# Layer: Plugins
RUN <plugin install commands>

# Layer: Zellij configuration (if zellij_config specified)
COPY zellij-config/ /home/coder/.config/zellij/
RUN cc-deck plugin install --force  # Overwrites cc-deck-managed files

# Layer: Project configuration (changes often)
COPY CLAUDE.md /home/coder/
COPY settings.json /home/coder/.claude/

USER coder
WORKDIR /home/coder
CMD ["/bin/zsh"]
```

### Step 4: Handle settings

- If `settings.zellij_config` is `current`: note that `cc-deck build` will copy `~/.config/zellij/` to the build context
- If `settings.zellij_config` is `vanilla`: skip the Zellij config layer
- If `settings.claude_md` is set: add COPY for the CLAUDE.md file
- If `settings.hooks` is set: add COPY for the hooks JSON

### Step 5: Optimize layers

- Order layers from least-frequently-changed to most-frequently-changed
- Combine related `dnf install` commands into a single RUN
- Clean package caches in the same layer as installs
- Use multi-stage builds if beneficial (e.g., compiling from source)

### Step 6: Write the file

Write the generated Containerfile to the build directory root.

### Key Rules

- Always include the "DO NOT EDIT" header with regeneration instructions
- Never include credentials or secrets in the Containerfile
- The cc-deck binary is copied from the build context (placed there by `cc-deck build`)
- Use `${TARGETARCH}` for multi-arch GitHub release downloads
- All github_tools entries must have download instructions
