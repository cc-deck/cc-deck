= Multi-User Deployment

Each developer gets their own cc-deck instance with individual credentials and persistent storage.

== One Deployment Per User

Since each user may need different authentication (API key, Vertex AI, different projects), the recommended pattern is one Deployment per user:

[source,bash]
----
# Create a namespace per user (or a shared namespace with naming conventions)
kubectl create namespace cc-deck

# User-specific Secret
kubectl -n cc-deck create secret generic alice-credentials \
  --from-literal=ANTHROPIC_API_KEY=sk-ant-alice-...

# User-specific Deployment
kubectl -n cc-deck apply -f alice-deployment.yaml
----

Example Deployment for a single user:

[source,yaml]
----
apiVersion: apps/v1
kind: Deployment
metadata:
  name: cc-deck-alice
  namespace: cc-deck
spec:
  replicas: 1
  selector:
    matchLabels:
      app: cc-deck
      user: alice
  template:
    metadata:
      labels:
        app: cc-deck
        user: alice
    spec:
      containers:
        - name: cc-deck
          image: quay.io/cc-deck/cc-deck-demo:latest
          envFrom:
            - secretRef:
                name: alice-credentials
          volumeMounts:
            - name: home
              mountPath: /home/coder
      volumes:
        - name: home
          persistentVolumeClaim:
            claimName: cc-deck-alice-home
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: cc-deck-alice-home
  namespace: cc-deck
spec:
  accessModes: ["ReadWriteOnce"]
  resources:
    requests:
      storage: 10Gi
----

This gives each user their own credentials, their own PVC, and their own Pod.

== Templating with Kustomize

For teams, use Kustomize to generate per-user resources from a base:

[source,text]
----
k8s/
├── base/
│   ├── deployment.yaml    # Template with USER placeholder
│   ├── pvc.yaml
│   └── kustomization.yaml
└── users/
    ├── alice/
    │   ├── kustomization.yaml
    │   └── secret.yaml
    └── bob/
        ├── kustomization.yaml
        └── secret.yaml
----

Each user overlay sets the name prefix, their Secret, and any user-specific configuration.

== Resource Planning

Each cc-deck Pod requires:

[cols="1,1,2"]
|===
| Resource | Minimum | Recommended

| Memory
| 4 Gi
| 8 Gi

| CPU
| 1 core
| 2-4 cores

| Storage (home)
| 5 Gi
| 10 Gi

| Storage (projects)
| 10 Gi
| 50 Gi
|===

For a team of 10 developers, plan for approximately 40 Gi of memory and 20-40 CPU cores.

== Shared Project Storage

For teams working on the same codebase, mount a shared ReadWriteMany volume alongside the per-user home:

[source,yaml]
----
volumes:
  - name: home
    persistentVolumeClaim:
      claimName: cc-deck-alice-home     # Per-user
  - name: shared-projects
    persistentVolumeClaim:
      claimName: team-projects          # ReadWriteMany, shared
----

Each user has their own home directory (Zellij config, Claude Code history) but shares project files.

== Connecting

Users connect to their own Pod:

[source,bash]
----
# Find your Pod
kubectl -n cc-deck get pods -l user=alice

# Connect
kubectl -n cc-deck exec -it deploy/cc-deck-alice -- zellij --layout cc-deck
----

== Cleanup

Remove a user's resources:

[source,bash]
----
kubectl -n cc-deck delete deployment cc-deck-alice
kubectl -n cc-deck delete pvc cc-deck-alice-home
kubectl -n cc-deck delete secret alice-credentials
----

WARNING: Deleting a PVC permanently removes the user's home directory and session data.
