= Kubernetes Credentials

Authentication for Claude Code in Kubernetes uses Secrets for API keys and service accounts for Vertex AI.

== API Key via Secret

[source,bash]
----
kubectl -n cc-deck create secret generic claude-credentials \
  --from-literal=ANTHROPIC_API_KEY=sk-ant-...
----

Reference in the StatefulSet:

[source,yaml]
----
envFrom:
  - secretRef:
      name: claude-credentials
----

== Vertex AI via Workload Identity

For Google Cloud Vertex AI on GKE, use Workload Identity Federation:

1. Create a Google Cloud service account with Vertex AI access
2. Bind it to the Kubernetes service account:
+
[source,bash]
----
gcloud iam service-accounts add-iam-policy-binding \
  cc-deck-sa@your-project.iam.gserviceaccount.com \
  --role roles/iam.workloadIdentityUser \
  --member "serviceAccount:your-project.svc.id.goog[cc-deck/cc-deck]"
----

3. Annotate the Kubernetes service account:
+
[source,yaml]
----
apiVersion: v1
kind: ServiceAccount
metadata:
  name: cc-deck
  namespace: cc-deck
  annotations:
    iam.gke.io/gcp-service-account: cc-deck-sa@your-project.iam.gserviceaccount.com
----

4. Set the Vertex environment variables in the StatefulSet:
+
[source,yaml]
----
env:
  - name: CLAUDE_CODE_USE_VERTEX
    value: "1"
  - name: CLOUD_ML_REGION
    value: us-east5
  - name: ANTHROPIC_VERTEX_PROJECT_ID
    value: your-project-id
----

== Vertex AI via Mounted Service Account Key

For non-GKE clusters, mount a service account key file:

[source,bash]
----
kubectl -n cc-deck create secret generic gcloud-key \
  --from-file=key.json=/path/to/service-account-key.json
----

[source,yaml]
----
env:
  - name: CLAUDE_CODE_USE_VERTEX
    value: "1"
  - name: CLOUD_ML_REGION
    value: us-east5
  - name: ANTHROPIC_VERTEX_PROJECT_ID
    value: your-project-id
  - name: GOOGLE_APPLICATION_CREDENTIALS
    value: /etc/gcloud/key.json
volumeMounts:
  - name: gcloud-key
    mountPath: /etc/gcloud
    readOnly: true
volumes:
  - name: gcloud-key
    secret:
      secretName: gcloud-key
----

== Additional Credentials

For MCP servers and other tools, add environment variables to the Secret:

[source,bash]
----
kubectl -n cc-deck create secret generic claude-credentials \
  --from-literal=ANTHROPIC_API_KEY=sk-ant-... \
  --from-literal=GITHUB_TOKEN=ghp_...
----
