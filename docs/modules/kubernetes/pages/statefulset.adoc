= StatefulSet Patterns

A Deployment with a PersistentVolumeClaim is the recommended pattern for running cc-deck on Kubernetes.
The PVC preserves your home directory (Zellij sessions, Claude Code history, snapshots) across Pod restarts.

== Deployment with PVC

[source,yaml]
----
apiVersion: apps/v1
kind: Deployment
metadata:
  name: cc-deck
spec:
  replicas: 1
  selector:
    matchLabels:
      app: cc-deck
  template:
    metadata:
      labels:
        app: cc-deck
    spec:
      containers:
        - name: cc-deck
          image: quay.io/cc-deck/cc-deck-demo:latest
          volumeMounts:
            - name: home
              mountPath: /home/coder
      volumes:
        - name: home
          persistentVolumeClaim:
            claimName: cc-deck-home
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: cc-deck-home
spec:
  accessModes: ["ReadWriteOnce"]
  resources:
    requests:
      storage: 10Gi
----

The PVC persists across Pod restarts, preserving your Zellij sessions, Claude Code history, and snapshots.

== Production StatefulSet

[source,yaml]
----
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: cc-deck
  namespace: cc-deck
spec:
  serviceName: cc-deck
  replicas: 1
  selector:
    matchLabels:
      app: cc-deck
  template:
    metadata:
      labels:
        app: cc-deck
    spec:
      securityContext:
        runAsUser: 1000
        runAsGroup: 1000
        fsGroup: 1000
      containers:
        - name: cc-deck
          image: quay.io/cc-deck/cc-deck-demo:latest
          envFrom:
            - secretRef:
                name: claude-credentials
          resources:
            requests:
              memory: "4Gi"
              cpu: "2"
            limits:
              memory: "8Gi"
              cpu: "4"
          volumeMounts:
            - name: home
              mountPath: /home/coder
            - name: projects
              mountPath: /home/coder/projects
  volumeClaimTemplates:
    - metadata:
        name: home
      spec:
        accessModes: ["ReadWriteOnce"]
        resources:
          requests:
            storage: 10Gi
    - metadata:
        name: projects
      spec:
        accessModes: ["ReadWriteOnce"]
        resources:
          requests:
            storage: 50Gi
----

== PersistentVolume for Source Code

For shared source code across team members, use a ReadWriteMany PV backed by NFS or a CSI driver:

[source,yaml]
----
apiVersion: v1
kind: PersistentVolume
metadata:
  name: team-projects
spec:
  capacity:
    storage: 100Gi
  accessModes:
    - ReadWriteMany
  nfs:
    server: nfs.internal
    path: /exports/projects
----

Each Pod mounts this shared volume at `/home/coder/projects`.

== Headless Service

The StatefulSet requires a headless Service for DNS resolution:

[source,yaml]
----
apiVersion: v1
kind: Service
metadata:
  name: cc-deck
  namespace: cc-deck
spec:
  clusterIP: None
  selector:
    app: cc-deck
  ports:
    - port: 8080
      name: http
----

This enables DNS like `cc-deck-0.cc-deck.cc-deck.svc.cluster.local`.
