= Credentials

Claude Code needs authentication to communicate with the Anthropic API.
cc-deck supports two methods: API key and Vertex AI.

== API Key Authentication

Pass your API key as an environment variable:

[source,bash]
----
podman run -d --name mydev \
  -e ANTHROPIC_API_KEY=sk-ant-... \
  quay.io/cc-deck/cc-deck-demo:latest
----

For security, you can read the key from a file instead of the command line:

[source,bash]
----
podman run -d --name mydev \
  -e ANTHROPIC_API_KEY=$(cat ~/.anthropic-key) \
  quay.io/cc-deck/cc-deck-demo:latest
----

== Vertex AI Authentication

For Google Cloud Vertex AI, mount your gcloud credentials and set the Vertex environment variables:

[source,bash]
----
podman run -d --name mydev \
  -e CLAUDE_CODE_USE_VERTEX=1 \
  -e CLOUD_ML_REGION=us-east5 \
  -e ANTHROPIC_VERTEX_PROJECT_ID=your-project-id \
  -v ~/.config/gcloud:/home/coder/.config/gcloud:ro \
  quay.io/cc-deck/cc-deck-demo:latest
----

The gcloud directory is mounted read-only (`:ro`) since the container only needs to read the credentials, not modify them.

=== Required Environment Variables

[cols="1,3"]
|===
| Variable | Description

| `CLAUDE_CODE_USE_VERTEX`
| Set to `1` to enable Vertex AI mode

| `CLOUD_ML_REGION`
| The Google Cloud region (e.g., `us-east5`, `europe-west1`)

| `ANTHROPIC_VERTEX_PROJECT_ID`
| Your Google Cloud project ID
|===

=== Refreshing Credentials

If your gcloud token expires, refresh it on the host:

[source,bash]
----
gcloud auth application-default login
----

The container picks up the new token automatically since the credentials directory is mounted.

== Environment File

For many environment variables, use an env file:

[source,bash]
----
# Create .env file (never commit this)
cat > .env << 'EOF'
ANTHROPIC_API_KEY=sk-ant-...
GITHUB_TOKEN=ghp_...
EOF

podman run -d --name mydev \
  --env-file .env \
  quay.io/cc-deck/cc-deck-demo:latest
----

This keeps credentials out of your shell history.
