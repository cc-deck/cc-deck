= CLI Reference

The `cc-deck` command-line tool manages the plugin, sessions, snapshots, and environment setup.

== Global Flags

[cols="1,1,3"]
|===
| Flag | Short | Description

| `--config`
|
| Config file path (default: `$XDG_CONFIG_HOME/cc-deck/config.yaml`)

| `--kubeconfig`
|
| Path to kubeconfig (default: `$KUBECONFIG` or `~/.kube/config`)

| `--namespace`
| `-n`
| Target Kubernetes namespace

| `--output`
| `-o`
| Output format: `text` (default), `json`, `yaml`

| `--profile`
| `-p`
| Credential profile to use

| `--verbose`
| `-v`
| Verbose output
|===

== Command Groups

Help output organizes commands into four groups by usage frequency.

Daily::
`attach`, `list` (`ls`), `status`, `start`, `stop`, `logs`.
These commands are available both at the top level and under `env`.

Session::
`snapshot`

Environment::
`env` (full lifecycle management)

Setup::
`plugin`, `profile`, `domains`, `setup`

== Daily Commands

Six high-frequency environment commands are available directly at the top level.
They behave identically to their `env` subcommand counterparts.

=== attach

Attach to a running environment.
For container environments, this opens a Zellij session inside the container.
Stopped containers are auto-started before attaching.
When no name is provided, resolves from `.cc-deck/environment.yaml` in the project.

[source,bash]
----
cc-deck attach mydev
cc-deck attach                        # resolves from project config
cc-deck attach --branch feature       # land in a specific worktree
----

[cols="1,3"]
|===
| Flag | Description

| `--branch`
| Attach and land in a specific worktree directory inside the container
|===

=== list

List all environments with their current status.

[source,bash]
----
cc-deck list
cc-deck ls                            # alias
cc-deck list --type container
cc-deck list --worktrees
cc-deck list -o json
----

[cols="1,1,3"]
|===
| Flag | Short | Description

| `--type`
| `-t`
| Filter by environment type

| `--worktrees`
| `-w`
| Show git worktrees within each project
|===

=== status

Display detailed status information for an environment.

[source,bash]
----
cc-deck status mydev
cc-deck status                        # resolves from project config
cc-deck status -o json
----

=== start

Start a stopped environment.

[source,bash]
----
cc-deck start mydev
cc-deck start                         # resolves from project config
----

=== stop

Gracefully stop a running environment.

[source,bash]
----
cc-deck stop mydev
cc-deck stop                          # resolves from project config
----

=== logs

View environment logs.

[source,bash]
----
cc-deck logs mydev
----

== Setup Commands

=== version

Print version information.

[source,bash]
----
cc-deck version
cc-deck version -o json
----

=== plugin

Manage the Zellij plugin installation.

==== plugin install

Install the WASM plugin, layouts, and Claude Code hooks.

[source,bash]
----
cc-deck plugin install
cc-deck plugin install --layout minimal
cc-deck plugin install --force
cc-deck plugin install --install-zellij
----

[cols="1,3"]
|===
| Flag | Description

| `--force`, `-f`
| Overwrite without prompting

| `--skip-backup`
| Skip creating backup of `settings.json`

| `--layout`
| Default layout variant: `standard` (default), `minimal`, `clean`

| `--install-zellij`
| Download and install matching Zellij binary
|===

==== plugin status

Show current installation status.

[source,bash]
----
cc-deck plugin status
cc-deck plugin status -o json
----

==== plugin remove

Remove the plugin, layouts, and hooks.

[source,bash]
----
cc-deck plugin remove
cc-deck plugin remove --skip-backup
----

=== snapshot

Manage session state snapshots.

==== snapshot save

Save current session state.

[source,bash]
----
cc-deck snapshot save my-checkpoint
cc-deck snapshot save --auto   # sequential naming
----

==== snapshot restore

Restore a saved snapshot.

[source,bash]
----
cc-deck snapshot restore my-checkpoint
----

==== snapshot list

List all saved snapshots.

[source,bash]
----
cc-deck snapshot list
cc-deck snapshot list -o json
----

==== snapshot remove

Delete a snapshot.

[source,bash]
----
cc-deck snapshot remove my-checkpoint
----

=== setup

Unified setup lifecycle commands for provisioning container images and SSH hosts from a declarative manifest.
The `--target` flag selects which provisioning targets to operate on.
Valid targets are `container`, `ssh`, or both as `container,ssh`.

==== setup init

Scaffold a setup directory with manifest and AI commands.
When no directory is specified and the project has `.cc-deck/`, defaults to `.cc-deck/setup/`.

[source,bash]
----
cc-deck setup init my-team-setup
cc-deck setup init                    # defaults to .cc-deck/setup/
cc-deck setup init my-team-setup --force
cc-deck setup init --target container
cc-deck setup init --target ssh
cc-deck setup init --target container,ssh
----

[cols="1,3"]
|===
| Flag | Description

| `--force`, `-f`
| Overwrite existing setup directory

| `--target`
| Provisioning targets to generate: `container`, `ssh`, or `container,ssh` (default: `container`)
|===

==== setup verify

Smoke-test a provisioned target.
For container targets, starts a container and checks tool availability, Claude Code startup, and cc-deck version.
For SSH targets, connects to the remote host and validates installed tools.
Reports pass/fail for each check.

[source,bash]
----
cc-deck setup verify my-team-setup --target container
cc-deck setup verify my-team-setup --target ssh
----

[cols="1,3"]
|===
| Flag | Description

| `--target`
| Target to verify: `container` or `ssh` (required)
|===

==== setup diff

Show manifest drift between the declared setup and the generated artifacts.
When `--target` is omitted, auto-detects available targets in the setup directory.

[source,bash]
----
cc-deck setup diff my-team-setup
cc-deck setup diff my-team-setup --target container
cc-deck setup diff my-team-setup --target ssh
----

[cols="1,3"]
|===
| Flag | Description

| `--target`
| Target to diff: `container` or `ssh` (auto-detected if omitted)
|===

Reports tools, plugins, MCP servers, and other components that are in the manifest but not reflected in the generated provisioning artifacts.

=== profile

Manage credential profiles for AI backends.

==== profile add

Interactively create a new credential profile.

[source,bash]
----

----

==== profile list

List all credential profiles.

[source,bash]
----
cc-deck profile list

[source,bash]

----

==== profile show

----

=== domains

Manage domain groups for network filtering in containerized sessions.
Domain groups are named collections of domain patterns (e.g., `python` includes `pypi.org` and related hosts).
Groups can be built-in, user-defined, or extended from built-in definitions.

==== domains init

Seed `~/.config/cc-deck/domains.yaml` with commented built-in group definitions as a starting point for customization.

[source,bash]
----
cc-deck domains init
cc-deck domains init --force
----

[cols="1,3"]
|===
| Flag | Description

| `--force`
| Overwrite existing `domains.yaml` file
|===

==== domains list

List all available domain groups with their source and domain count.
Sources include `builtin` (shipped with cc-deck), `user` (defined in `domains.yaml`), and `extended` (user config that extends a built-in group).

[source,bash]
----
cc-deck domains list
----

==== domains show

Display the expanded domain patterns for a specific group.

[source,bash]
----
cc-deck domains show python
cc-deck domains show github
----

==== domains blocked

Show blocked requests from the tinyproxy logs of a running Podman session.
This command requires Podman and a session deployed with network filtering enabled.

[source,bash]
----
cc-deck domains blocked my-session
cc-deck domains blocked my-session --since 30m
----

[cols="1,3"]
|===
| Flag | Description

| `--since`
| Show blocks from the last duration (default: `1h`)
|===

==== domains add

Add a domain or domain group to a running Podman session.
The proxy container is restarted to apply the change.

[source,bash]
----
cc-deck domains add my-session pypi.org
cc-deck domains add my-session rust
----

==== domains remove

Remove a domain or domain group from a running Podman session.
The proxy container is restarted to apply the change.

[source,bash]
----
cc-deck domains remove my-session pypi.org
cc-deck domains remove my-session docker
----

== Environment Commands

=== env

Manage environments across all supported backends (local, container, compose).
When run inside a project with `.cc-deck/environment.yaml`, environment names are resolved automatically.

NOTE: The commands `attach`, `list`, `status`, `start`, `stop`, and `logs` are also available at the top level (see <<Daily Commands>>).

==== env create

Create a new environment.
When run inside a project with `.cc-deck/environment.yaml`, name and settings are read from the definition.
In a git repository without a definition, one is scaffolded from CLI flags before provisioning.

[source,bash]
----
cc-deck env create                    # reads from .cc-deck/environment.yaml
cc-deck env create mydev --type local
cc-deck env create --type compose --image quay.io/cc-deck/cc-deck-demo:latest
cc-deck env create --variant auth     # separate container instance
cc-deck env create mydev --type container --port 8080:80
----

[cols="1,1,3"]
|===
| Flag | Short | Description

| `--type`
| `-t`
| Environment type: `local`, `container`, `compose`, `ssh`, `k8s-deploy`, `k8s-sandbox`

| `--image`
|
| Container image to use (container, compose)

| `--port`
| `-p`
| Port mapping (host:container), repeatable (container, compose)

| `--all-ports`
|
| Expose all container ports (container)

| `--storage`
|
| Storage type: `named-volume` (container default), `host-path` (compose default)

| `--path`
|
| Project directory (compose: defaults to cwd)

| `--credential`
|
| Credential as `KEY=VALUE`, repeatable (container, compose)

| `--mount`
|
| Bind mount as `src:dst[:ro]`, repeatable (container, compose)

| `--auth`
|
| Auth mode: `auto` (default), `none`, `api`, `vertex`, `bedrock` (container, compose, ssh)

| `--allowed-domains`
|
| Domain groups for network filtering, repeatable (compose only)

| `--variant`
|
| Variant name for multiple container instances from the same definition

| `--host`
|
| SSH target host as `user@hostname` (ssh only)

| `--ssh-port`
|
| SSH port (ssh only, default: 22)

| `--identity-file`
|
| Path to SSH private key (ssh only)

| `--jump-host`
|
| SSH jump/bastion host (ssh only)

| `--ssh-config`
|
| Custom SSH config file path (ssh only)

| `--workspace`
|
| Remote workspace directory (ssh only, default: `~/workspace`)
|===

[source,bash]

==== env delete

Delete an environment and its resources.
For project-local environments, removes runtime state (`status.yaml`, `run/`) but preserves the committed definition (`environment.yaml`).

[source,bash]
----
cc-deck env delete mydev
cc-deck env delete                    # resolves from project config
cc-deck env delete --force --keep-volumes
----

[cols="1,1,3"]
|===
| Flag | Short | Description

| `--force`
| `-f`
| Force delete a running environment

| `--keep-volumes`
|
| Preserve data volumes when deleting container environments
|===

==== env list

List all environments with their current status.
Shows both global and project-local environments in a unified view.
Projects registered in the global registry are included with their paths.

[source,bash]
----
cc-deck env list
cc-deck env list --type container
cc-deck env list --worktrees
cc-deck env list -o json
----

|===

| `-w`

[source,bash]

==== env start

Start a stopped environment.

cc-deck env start mydev

==== env stop

Gracefully stop a running environment.

[source,bash]

cc-deck env stop mydev
cc-deck env stop                      # resolves from project config

==== env logs

View environment logs.

[source,bash]
----

----

==== env refresh-creds

Push fresh credentials to a remote SSH environment without attaching.
This is useful for keeping long-running sessions alive when local credentials rotate.
Only applicable to SSH environments.

[source,bash]
----
cc-deck env refresh-creds remote-dev
cc-deck env refresh-creds             # resolves from project config
----

==== env prune

Remove stale entries from the global project registry.
Entries whose directories no longer exist are removed.

[source,bash]
----
cc-deck env prune
----

==== env exec

Run a command inside a container environment.

[source,bash]
----
cc-deck env exec mydev -- ls /workspace
cc-deck env exec mydev -- npm install
----

==== env push

Copy local files into a container environment.

[source,bash]
----
cc-deck env push mydev ./src /workspace/src
cc-deck env push mydev ./config.yaml
----

==== env pull

Copy files from a container environment to the local machine.

[source,bash]
----
cc-deck env pull mydev /workspace/output ./output
cc-deck env pull mydev /workspace/results
----

==== env harvest

Extract work products from an environment.
Not supported for container environments (use push/pull instead).

[source,bash]
----
cc-deck env harvest mydev
----

=== hook

Internal command used by Claude Code hooks.
Not intended for direct use.

[source,bash]
----
cc-deck hook --pane-id "$ZELLIJ_PANE_ID"
----

Forwards Claude Code session events to the sidebar plugin.
